## 4. Despliegue completo en una máquina virtual (Ubuntu Server + Apache)

Se armó y probó de principio a fin el entorno que va a correr SICET en
producción, dentro de una VM de VirtualBox (para eventualmente subirse a un
servidor físico o quedarse como está).

- **VM:** Ubuntu Server 24.04.4 LTS, red en modo **Adaptador Puente** (no
  NAT) sobre la tarjeta WiFi del host, para que la VM quede en la misma LAN
  que el servidor de base de datos y sea alcanzable por el resto de la
  planta. IP asignada por DHCP: `192.168.10.145`.
- **Stack instalado:** Apache2 + `libapache2-mod-php8.3` (con `mpm_prefork`,
  requerido por `mod_php`), PHP 8.3 con sus extensiones (`mysql`, `mbstring`,
  `xml`, `bcmath`, `curl`, `zip`, `gd`), Composer, Node.js 20 + npm, Git,
  cliente de MySQL.
- **Código:** clonado desde GitHub (rama `produccion`) usando una llave SSH
  de despliegue de solo lectura (el repo es privado). `composer install`
  necesitó `--ignore-platform-reqs`: `composer.json` exige `ext-oci8`
  (paquete `yajra/laravel-oci8`) aunque esa conexión Oracle nunca se usa (la
  app corre sobre `mysql`) — sin esa bandera, la instalación falla.
- **`.env`:** copiado desde el de desarrollo y ajustado para producción
  (`APP_ENV=production`, `APP_DEBUG=false`, `APP_URL`, `MAIL_REDIRECT_TO`
  se deja puesto por ahora — ver Pendientes).
- **Bugs de arranque encontrados y resueltos, en orden:**
  1. Faltaba `php8.3-xml` (Apache se instaló a medias por una caída temporal
     del mirror `security.ubuntu.com`) → error `Class "DOMDocument" not
     found` en `composer install`.
  2. Faltaba `libapache2-mod-php8.3` → Apache servía el código fuente de
     `index.php` como texto plano en vez de ejecutarlo.
  3. Faltaba `php8.3-mysql` (`pdo_mysql`) → `PDOException: could not find
     driver` al intentar cualquier consulta.
  4. `storage/framework/sessions/` no existe en el repositorio (falta el
     placeholder que debería mantenerla en git) → `file_put_contents(...):
     No such file or directory` al iniciar sesión. Se creó a mano en la VM;
     **queda pendiente** agregar el placeholder al repo para que no le
     vuelva a pasar a nadie que clone desde cero.
- **Verificado funcionando:** login con `94` / `7410` carga correctamente
  contra la base de datos remota.
